# Document Brand Voice Guidelines as Fact

Create a fact document that captures Dust's brand voice and messaging guidelines. This will serve as a reference for anyone writing documentation, CLI output, error messages, or any other user-facing content.

## Context

The brand identity framework has established clear voice attributes and examples. These should be documented as a fact so agents and contributors can consistently apply them across all touchpoints.

Based on the resolved questions:
- **Brand Style**: Minimal, Tool-Focused (like a Unix utility)
- **Voice**: Direct, technical, practical, confident
- **Audience**: Developers using AI coding tools

This fact should be concise and actionable, focusing on practical examples rather than abstract descriptions.

## Principles

- lightweight-planning.md - Dust aims to be a minimal, low-overhead planning system
- clarity-over-brevity.md - Names should be descriptive and self-documenting
- small-units.md - Facts should be discrete and fine-grained

## Definition of Done

- [ ] Create `.dust/facts/brand-voice.md` with:
  - Core voice attributes (direct, technical, practical, confident)
  - 8-10 concrete examples of good vs. bad messaging
  - Specific guidance for different contexts (docs, CLI, errors)
  - Reference to resolved positioning decisions
- [ ] Keep the fact concise (under 100 lines)
- [ ] Include actionable guidance, not just descriptions
- [ ] Format follows existing fact file patterns
