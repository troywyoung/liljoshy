# Build How It Works Section

Implement the "How It Works" section. Show the four artifact types (tasks, ideas, principles, facts) and the five-step workflow (capture, pick, implement, commit, repeat).

## Context

This section provides concrete understanding of Dust's workflow. It should use progressive disclosure to show high-level workflow first, with the ability to explore artifact types in more detail.

## Requirements

- Section headline: "Four artifact types, one workflow"
- Display four artifact types:
  1. **Tasks** - Actionable work with definitions of done
  2. **Ideas** - Proposals waiting to be refined
  3. **Principles** - Guiding values and design constraints
  4. **Facts** - Documentation of current system state
- Display five-step workflow:
  1. Capture → Create artifacts in `.dust/` as markdown
  2. Pick → Agent reads context and selects a task
  3. Implement → Agent makes changes defined in task
  4. Commit → Atomic commit including task deletion
  5. Repeat → Continue until backlog is clear
- Visual representation (diagram, animation, or structured layout)
- Progressive disclosure: summary visible, details expandable
- Follow brand voice: direct, technical, practical, confident
