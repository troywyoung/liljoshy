# Design product owner meta interface

Design the first internal interface for configuring app variants. Specify the minimum product-owner workflows for editing prompt structure, design system choices, user-facing copy, and duplicate-app creation so implementation can stay focused on the smallest useful admin surface.

## Task Type

implement

## Blocked By

- [Define Meta Application Model](define-meta-application-model.md)


## Definition of Done

- The core product-owner workflows are described end to end.
- The interface scope is limited to the minimum set needed to manage Hot Take and create one derivative app.
- The design addresses clarity, speed, and low-friction editing for a non-technical product owner.
- The design identifies which settings should be editable immediately and which should remain code-only for now.
