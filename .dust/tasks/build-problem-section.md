# Build Problem Section

Implement the problem section of the landing page. Explain three core challenges agents face: context loss, ambiguous requirements, and no quality control.

## Context

This section addresses the core objection "why do I need this?" by explaining the problems that Dust solves. Each problem needs a short title, pull quote, and body paragraph following the brand voice.

## Requirements

- Section headline: "Why agents keep letting you down" (or similar direct framing)
- Three problem cards:
  1. **Context Loss**: Every session starts from zero, agents forget requirements, manual context management doesn't scale
  2. **Ambiguous Requirements**: Without specs agents improvise, code compiles but misses the point, rework loops
  3. **No Quality Control**: Agents can't self-check, bugs ship because there are no gates, you become the QA bottleneck
- Each problem has:
  - Short title (3-5 words)
  - Pull quote (one sentence explaining the symptom)
  - Body paragraph (2-3 sentences on consequences)
- Follow brand voice: direct, technical, practical, confident
- No forbidden words or marketing speak

- Problem section renders with headline
- Three problem cards display correctly
- Each card has title, pull quote, and body paragraph
- Copy follows brand voice guidelines

- Section is responsive and accessible
