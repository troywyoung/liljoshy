# Build internal meta interface

Build the first internal product-owner interface for managing app variants. Let the owner edit the selected configuration fields directly in the browser and prepare the system for fast iteration on prompts, design, and messaging.

## Task Type

implement

## Blocked By

- [Design Product Owner Meta Interface](design-product-owner-meta-interface.md)
- [Implement Config Driven App Variants](implement-config-driven-app-variants.md)


## Definition of Done

- An internal interface exists for viewing and editing the selected configuration fields.
- The interface supports updating prompt structure, user messaging, and key design-system settings for an app variant.
- The interface supports creating a new app variant by duplicating an existing one and changing its identity fields.
- Changes made through the interface are persisted in the chosen storage approach and reflected by the runtime app.
