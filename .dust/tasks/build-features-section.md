# Build Features Section

Implement the feature deep dive section. Highlight five core features: task-first workflow, atomic commits, quality gates, progressive disclosure, and trunk-based development.

## Context

This section provides technical depth on Dust's core capabilities. Each feature needs clear explanation without marketing speak, showing what it does and why it matters.

## Requirements

- Section with five feature cards:
  1. **Task-First Workflow**: Every change starts as documented task, creates traceability
  2. **Atomic Commits**: Implementation + documentation + task deletion in one commit, git history tells complete story
  3. **Quality Gates**: `dust check` validates before commit, configurable checks (tests, lint, types)
  4. **Progressive Disclosure**: Minimal context in tasks, agent reads deeper as needed, stays within context windows
  5. **Trunk-Based Development**: No branching required, direct commits to main, fast feedback loops
- Each feature has:
  - Feature name
  - Two-sentence explanation: what it does, why it matters
- Follow brand voice: direct, technical, practical
- Link to relevant principles where appropriate
