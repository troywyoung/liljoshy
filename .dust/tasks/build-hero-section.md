# Build Hero Section

Implement the hero section of the Dust landing page with the headline "Stop managing agents. Start building.", two-sentence subheadline, primary and secondary CTAs, and a terminal mockup visual showing dust CLI output.

## Context

This is the first vertical slice of the landing page. The hero section needs to communicate what Dust is in under 10 seconds and provide immediate CTAs to GitHub and documentation.

## Requirements

- Hero section with headline: "Stop managing agents. Start building."
- Tagline: "Flow state for AI coding agents."
- Two-sentence subheadline explaining what Dust provides and what it enables
- Primary CTA linking to GitHub repository
- Secondary CTA linking to documentation
- Terminal mockup showing realistic dust CLI output (task list, agent workflow, or dust check output)
- Responsive design that works across devices
- Follow brand voice guidelines: direct, technical, practical, confident
- No forbidden words: revolutionize, unlock, empower, seamless, etc.

## Task Type

implement

(none)

## Definition of Done

- Hero section renders on landing page
- Headline, tagline, subheadline, and CTAs are present and follow brand voice
- Terminal mockup displays realistic dust CLI output
- All links work correctly (GitHub and documentation)
- Section is responsive and accessible
- No forbidden marketing terms used
- Page loads quickly with optimized assets
