# Build Social Proof Section

Implement social proof using GitHub stars and usage statistics, avoiding traditional testimonials to maintain the Unix-utility brand feel.

## Context

This section builds trust through quantitative, verifiable metrics rather than marketing testimonials. Fits the technical brand voice by showing objective data.

## Requirements

- Display GitHub stars count (dynamically fetched or static)
- Display usage statistics if available (npm downloads, active repositories, etc.)
- Open source badge
- MIT license notice
- Visual styling consistent with technical, minimal brand
- Optional: Link to GitHub repository for contributing docs
- Follow brand voice: technical, quantitative, no marketing fluff

## Task Type

implement

## Blocked By

(none)

## Definition of Done

- Social proof section renders on page
- GitHub stars displayed (static or dynamic)
- Usage statistics shown if available
- Open source badge and MIT license notice present
- Section styling is minimal and technical
- Links to GitHub work correctly
- Section is responsive and accessible
