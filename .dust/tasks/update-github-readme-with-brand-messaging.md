# Update GitHub README with Brand Messaging

Refine the Dust README to follow the established brand messaging framework. The README serves as the primary landing page for developers discovering Dust on GitHub and should clearly communicate what Dust does, why it matters, and how to get started.

## Context

Based on the resolved brand identity decisions:
- **Primary CTA Goal**: GitHub Stars & Community Growth
- **Positioning**: Alternative to Manual Agent Management
- **Business Model**: Freemium SaaS
- **Brand Style**: Minimal, Tool-Focused
- **Technical Depth**: Show the Code

The README should adopt a direct, technical voice that speaks developer-to-developer. Avoid hype and marketing language. Show real commands and workflows.

## Principles

- agent-autonomy.md - Dust exists to enable AI agents to produce work autonomously
- easy-adoption.md - Dust should be trivially easy to adopt
- unsurprising-ux.md - The user interface should be as "guessable" as possible
- clarity-over-brevity.md - Names should be descriptive and self-documenting

## Definition of Done

- [ ] README.md updated with new brand messaging structure:
  - Hero section with clear tagline and value proposition
  - Problem statement explaining why agents need structured workflows
  - Quick start with actual commands (npm install, dust init, etc.)
  - Core features section showing CLI commands and examples
  - Links to GitHub repo and documentation
- [ ] Messaging follows brand voice: direct, technical, practical, confident
- [ ] Primary CTA is "Get Started" with secondary "Star on GitHub"
- [ ] Code examples are prominent and accurate
- [ ] No marketing hype or buzzwords
- [ ] All links and commands are tested and working
